2. Abstraction: Simplify Complex Details

Abstraction exposes only what the user needs to know and hides internal complexity.

Example — Book

Students only see: title, author, category, availability. Internal details (publisher, edition, ISBN, shelf location) are hidden from student views.

Conceptual Java example:

// Book.java — simplified
class Book {
  private String title;
  private String author;
  private String category;
  private boolean available;

  // Internal details (hidden)
  private String ISBN;
  private String publisher;
  private String edition;

  // Public API — minimal
  public String getTitle(){ return title; }
  public String getAuthor(){ return author; }
  public String getCategory(){ return category; }
  public boolean isAvailable(){ return available; }
}

3. Information Hiding (Protect Sensitive Data)

Use access control, encapsulation, encryption and role checks to protect sensitive data.

Sensitive items

  • Librarian credentials — store hashed/encrypted, never reveal raw password
  • Student fines — viewable only to the student and authorized librarians
  • Borrow history — restricted to authorized roles

Conceptual Java snippets:

// Librarian.java — credentials hidden
class Librarian {
  private String username;
  private String encryptedPassword; // store only encrypted/hashed password

  public boolean login(String inputPassword){
    return encrypt(inputPassword).equals(encryptedPassword);

4. Summary

Key takeaways:

  • Abstraction — show only necessary fields to users, hide internal fields.
  • Information hiding — keep sensitive data private, enforce access control and store secrets securely.
  • Encapsulation — wrap data and expose only safe operations (e.g., borrowBook(), returnBook()).
